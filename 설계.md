## KETI AI Studio — 웹 기반 AI 책 제작 스튜디오 설계

서비스 정체성은 다음과 같습니다.

**자료를 분석하고, 페르소나를 적용해 책을 생성한 뒤, 직접 편집하여 PDF·DOCX·Markdown 또는 GitHub로 내보내는 웹 기반 AI 책 제작 스튜디오**

---

## 전체 메뉴(사이드바)

**KETI AI Studio**

- 홈
- 새 책 만들기
- 책 관리
- ────────────
- 페르소나 관리
- 실행 기록
- ────────────
- 설정

### 연동 상태(UI 표시 규칙)

- ● GitHub: 연결됨
- ○ Telegram: 미연결

현재 프로젝트 구조에는 Telegram Adapter가 없으므로 실제 UI에서는 다음 중 하나로 처리합니다.

- Telegram 메뉴를 당분간 숨김
- 또는 “미연결 · 준비 중”으로 표시

GitHub는 `backend/services/github_client.py`가 있으므로 **연결 검증에 성공한 경우에만** “연결됨”으로 표시합니다.

## 책 제작 순서(마법사 단계)

화면 상단의 책 생성 단계는 4개로 제한합니다.

**1 자료 → 2 설정·페르소나 → 3 목차 → 4 생성**

검토·수정·출판은 생성 마법사 이후 **책 편집기**에서 처리합니다.

## 2. 홈과 새 책 만들기 화면

### 2-1. 홈

홈에는 세 영역만 둡니다.

#### AI 빠른 요청

파일이나 링크를 올리고 분석·집필을 요청하세요.

`[파일 첨부] [URL 추가]                              [전송]`

지원 작업:

- 파일·URL 내용 분석
- 중요한 내용 요약
- 자료 기반 질의응답
- 새로운 책 제작 시작
- 기존 책 이어쓰기

사용자가 파일을 올리고 “중요한 내용 알려줘”라고 하면:

- `backend/api/sources.py`로 자료 등록
- `backend/services/extractors.py`로 내용 추출
- `backend/agents/research_agent.py`로 분석
- 핵심 주제·근거·부족한 내용 표시

“이 자료로 기술서 써줘”라고 하면 분석 결과 아래에 다음 버튼을 표시합니다.

- `[새 책으로 만들기]`

이 버튼을 누르면 분석된 자료를 유지한 채 `/books/new`로 이동합니다.

#### 최근 책

- 표지
- 제목
- 상태
- 진행률
- 마지막 수정일
- 이어서 편집

#### 최근 작업

별도 큰 패널을 만들지 않고 최근 작업 3건만 간단하게 표시합니다. 전체 작업은 실행 기록에서 확인합니다.

---

### 2-2. 새 책 만들기 1단계 — 자료

연결 코드:

- `backend/api/sources.py`
- `backend/services/extractors.py`
- `backend/agents/research_agent.py`

#### 왼쪽: 자료 추가

지원 파일:

- PDF, HWP, HWPX, DOCX, XLSX, CSV, MD, TXT

지원 입력:

- 파일 업로드
- URL 추가
- 등록 자료 삭제
- 분석 재실행
- 책에 포함/제외

자료 상태:

- 업로드 중 → 추출 중 → 분석 중 → 분석 완료 / 실패

#### 오른쪽: 분석 결과

- 핵심 주제
- 주요 근거
- 중요한 수치
- 부족한 내용
- 추천 책 방향
- 출처 위치

분석 결과는 문장만 보여주지 말고 출처를 함께 저장합니다.

예시:

```json
{
  "text": "ALD 공정에서 온도와 퍼지 시간이 중요합니다.",
  "source_id": 12,
  "location": "PDF 13~16쪽"
}
```

#### 하단 채팅(자료 범위 제한)

이 단계의 채팅 범위는 등록한 자료로 한정합니다.

현재 범위: 선택 자료 3개

예:

- 이 자료의 핵심은?
- 두 자료의 차이를 비교해줘.
- 시장 전망 부분만 정리해줘.
- 이 자료로 전문가용 책을 만들 수 있어?

### 2-3. 2단계 — 설정·페르소나

연결 코드:

- `backend/api/config.py`
- `backend/api/personas.py`
- `backend/agents/config_designer.py`
- `backend/services/persona_store.py`

#### 책 설정(직접 수정 가능)

- 책 제목
- 책 유형
- 대상 독자
- 책의 목적
- 전문성 수준
- 목차 개수
- 목차당 기본 글자 수
- 예상 전체 글자 수
- 참고문헌 사용
- 표·차트 사용
- 다이어그램 사용

AI가 `config_designer.py`로 먼저 추천값을 작성합니다.

사용자는 추천 후에도 모든 항목을 직접 수정할 수 있습니다.

추천 표시 예:

- AI 추천: 목차 8개 / 목차당 6,000자 / 예상 48,000자

#### 페르소나

- AI 추천 페르소나
- 다른 페르소나 선택
- 선택한 페르소나 수정
- 복제하여 새 페르소나 저장

페르소나를 수정해도 **원본 시스템 페르소나를 변경하지 않습니다.** 사용자 복사본을 만들어 `data/personas/`에 저장합니다.

### 2-4. 3단계 — 목차

연결 코드:

- `backend/api/outlines.py`
- `backend/api/units.py`
- `backend/agents/outline_planner.py`
- `backend/storage/recalculation.py`

각 목차(unit)는 다음 값을 가집니다.

- 순서
- 제목
- 설명
- 목표 글자 수
- 사용 자료
- 필수 포함 내용
- 예정 시각자료
- 상태

지원 작업:

- 목차 직접 수정
- 드래그 순서 변경
- 목차 추가/삭제
- 목차 개수 조절
- 장/절 구조 조절
- 목차별 글자 수 변경
- 특정 자료 연결
- 표·차트·다이어그램 계획 변경
- AI 목차 전체 재생성
- 선택 목차만 AI 재생성

`recalculation.py`는 다음 값을 자동 재계산합니다.

- 목차 개수
- 전체 예상 글자 수
- 전체 예상 페이지
- 시각자료 개수

목차 승인을 누르기 전에는 챕터를 생성하지 않습니다.

버튼:

- `[임시 저장] [목차 다시 생성] [목차 승인하고 생성 시작]`

## 5. 시각자료 설계

현재 존재하는 코드:

- `backend/api/visuals.py`
- `backend/agents/visual_planner.py`
- `backend/agents/diagram_agent.py`
- `backend/services/data_tools.py`

따라서 현재 지원 범위는 다음으로 잡아야 합니다.

- 표
- 데이터 차트
- 다이어그램
- 프로세스 흐름도

아직 `image_agent.py`가 없기 때문에 소설 삽화나 창의적 이미지 생성을 이미 구현된 것처럼 표시하면 안 됩니다.

실행 구조(예):

`visual_planner.py`가 시각자료의 필요성과 유형을 결정합니다.

```json
{
  "unit_id": 23,
  "visual_type": "chart",
  "purpose": "온도 변화에 따른 수율 비교",
  "source_ids": [4],
  "status": "planned"
}
```

책 유형별 기본 정책(현재 가능한 범위):

- 전문가 기술서: 표, 차트, 공정도, 구조도
- 데이터 분석서: 통계표, 막대·선·분포 차트
- 교육서: 개념도, 단계도, 비교표
- 비즈니스 보고서: KPI 차트, 비교표, 로드맵
- 소설: 장면 구분, 등장인물 표만 가능
- 창의적 삽화: 추후 Image Agent 추가 필요

사용자는 목차 화면과 편집기에서 시각자료를 수정·삭제·재생성할 수 있습니다.

### 7-2. 실행 기록(추가 권장)

현재 디렉터리에는 `runs.py` 또는 작업 기록 전용 API가 없으므로, 실행 기록 화면을 제대로 구현하려면 아래 추가를 권장합니다.

```text
backend/
├── api/
│   └── runs.py
├── services/
│   └── run_tracker.py
└── storage/
    └── models.py
        ├── Run
        └── RunStep
```

모델 예:

- Run: id, book_id, action, status, progress, started_at, finished_at, error_message, result_artifact_id
- RunStep: run_id, step_name, status, started_at, finished_at, summary

- 자료 분석

- 검토
- 시각자료 생성
- PDF 생성
- GitHub 업로드

Agent 이름·프롬프트·원시 응답은 개발자 모드에서만 표시합니다.
